Sponsorship Workflow

Sponsorships follow this lifecycle: Applied → Approved → Finalized (or Rejected)

Key Actions
  • Review — Click any sponsorship from the Sponsorships tab to see full details
  • Approve — Set start/end dates, fee, and package. Creates a draft contract. Use quick-date buttons for speed
  • Approve with Signed Contract — Upload a pre-signed contract during approval (skips generate/send)
  • Reject — Sponsor gets notified by email
  • Rollback — Returns an approved/rejected sponsorship to "Applied" for corrections
  • Lock/Unlock — Controls whether a finalized sponsorship can be edited

Composer (Package Builder)

The Composer is a 5-step wizard for building custom sponsorship packages:

  1. Select Sponsor — Search existing or create new
  2. Choose Package — Pick a base tier or go custom/a-la-carte
  3. Customize Benefits — Add/remove benefits. Package benefits are locked (blue). Pkg-only benefits warn before adding
  4. Set Terms — Fee (with match/adjust buttons), dates, renewal flag, notes
  5. Review & Send — Create the sponsorship, email internally for review, or send proposal to sponsor

Contract Management

Contracts flow: Draft → Awaiting Signature → Executed

  • Generate — Renders PDF/DOCX from the contract template
  • Send — Email to sponsor contacts or internal review (any @pyfound.org address)
  • Upload Signed — Upload the signed PDF to execute and finalize
  • Nullify — Void a contract. Can re-draft afterward
  • Preview/Download — PDF and DOCX available at any stage

Benefits & Packages

  • Benefits — Create/edit benefits, manage feature configurations (logo placements, tiered quantities, required assets)
  • Packages — Create/edit sponsorship tiers (Visionary, Sustainability, etc.)
  • Clone Year — Copy all packages and benefits from one year to another

Bulk Actions & Export

  • CSV Export — Click "Export CSV" on the sponsorships page, or select rows and use bulk action
  • Asset Export — Download all submitted assets as ZIP from sponsorship detail or via bulk action
  • Bulk Notify — Select multiple sponsorships and send a notification to all their contacts

Sponsor Contacts

Each sponsor can have multiple contacts with roles: Primary, Administrative, Accounting, Manager. Edit contacts from the sponsorship detail page's Sponsor card.

Still in Django Admin

These tasks still require Django Admin:

  • Benefit ordering (drag to reorder)
  • Benefit conflict configuration
  • Legal clause management
  • Sponsorship program management (Foundation, PyCon, PyPI, etc.)
  • Advanced import/export with django-import-export
  • Detailed asset filtering and administration
